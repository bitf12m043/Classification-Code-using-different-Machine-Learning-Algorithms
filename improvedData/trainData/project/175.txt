Date Tue Nov GMT Server NCSA Content type text html Last modified Sat Nov GMT Content length Lazy Receiver Processing LRP LRP LRP new network subsystem architecture based Lazy Receiver Processing LRP motivated the poor performance that interrupt driven network subsystems show under heavy load Such systems give strictly highest priority packet processing and not correctly account for resources consumed during protocol processing Under conditions heavy network load this leads unfair allocation system resources scheduling anomalies and livelock LRP properly integrates network processing with the operating system resource allocation scheme The resources spent protocol processing are accounted for and charged the application whose behalf this processing performed The system combines early packet demultiplexing early discard excess packets and lazy processing model where packet processing performed the receiver priority and just time avoid delaying the receiving application This enables the system provide stable overload behaviour fair resource allocation and increased throughput under heavy network load prototype implementation LRP was constructed modifying the TCP UDP network subsystem the BSD Lite distribution optionally implement LRP new protocol family The prototype was tested and evaluated ATM network SparcStation downloading this code into the SunOS kernel loadable kernel module Experiments this testbed show that under high network load the LRP system dramatically more stable and yields higher throughput than the BSD network system without any performance degradation under normal conditions plan release our source code soon licensing issues have been resolved For more details about LRP refer extended version our OSDI paper html the conference version postscript contact gaurav rice edu druschel rice edu About the daemon image above
